# 渠道与分组

对普通用户来说，最重要的不是“渠道”本身，而是：

1. 你的 **API Key**
2. Key 绑定的 **分组**
3. 分组开放的 **模型列表**（例如 `grok-4.5`）

## 用户侧怎么理解

```text
客户端  --API Key-->  Arona 中转站  --分组路由-->  上游账号池
```

你只需要：

- 在控制台创建 Key
- 确认 Key 能调用 `grok-4.5`
- 把 Base URL 指到 `https://sub2api.lingshichat.top/v1`

## 控制台里建议检查的项

- 密钥是否启用
- 分组是否包含 Grok 模型
- 余额 / 订阅是否可用
- 是否触发风控或限流

## 管理员侧（简述）

管理员可配置上游渠道、账号池、模型映射与负载均衡。普通接入用户通常无需处理这些细节。
