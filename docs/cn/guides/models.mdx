# 模型支持

## 当前推荐

| 模型 ID | 说明 | 建议客户端 |
| --- | --- | --- |
| `grok-4.5` | 主推对话 / 编程模型 | Grok Build、Codex、Claude Code、OpenCode、Pi |

文档后续会补充其他模型；**请始终以你账号实际返回的模型列表为准**。

## Grok 4.5 上下文（重要）

| 项 | 值 |
| --- | --- |
| 模型硬上限 | 约 **500k** tokens |
| **质量推荐上限** | 约 **200k** tokens |
| 客户端配置建议 | `context_window` / `contextWindow` = **`200000`** |

<Warning>
  **虽然 Grok 4.5 支持约 500k 上下文，但超过约 200k 会明显降智。**  
  编程代理请把实用窗口压在 **200k 以内**，让客户端更早自动压缩；不要为「占满 500k」长期堆超长会话。详见 [Grok Build 配置 · 上下文](/cn/clients/grok-build#上下文配置务必读)。
</Warning>

## 查询你的可用模型

```bash
curl https://sub2api.lingshichat.top/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

返回示例（结构示意）：

```json
{
  "object": "list",
  "data": [
    {
      "id": "grok-4.5",
      "object": "model",
      "owned_by": "xai"
    }
  ]
}
```

## 模型与分组

- API Key 绑定的 **分组** 决定能看到哪些模型
- 同一模型名可能映射到不同上游账号池
- 若 `model_not_found`，先检查分组，而不是只改客户端

## 调用示例

```bash
curl https://sub2api.lingshichat.top/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grok-4.5",
    "messages": [{"role": "user", "content": "你好"}]
  }'
```
