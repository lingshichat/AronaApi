# 快速开始

本指南帮助你完成：注册 → 创建 Key → 验证接口 → 配置客户端。

## 前置条件

- 一个 [Arona API 控制台](https://sub2api.lingshichat.top) 账号
- 可用的 API Key（密钥页创建）
- 想要接入的客户端：Grok Build / Codex / Claude Code

## 1. 获取 API Key

1. 登录 [https://sub2api.lingshichat.top](https://sub2api.lingshichat.top)
2. 进入 **API Keys / 密钥** 页面
3. 创建新密钥并复制保存

<Warning>
  API Key 只显示一次或有限次，请勿提交到公开仓库或发给不信任的第三方。
</Warning>

## 2. 确认连接信息

| 配置项 | 值 |
| --- | --- |
| 控制台 | `https://sub2api.lingshichat.top` |
| API Base URL | `https://sub2api.lingshichat.top/v1` |
| 推荐模型 | `grok-4.5` |
| 鉴权头 | `Authorization: Bearer <YOUR_API_KEY>` |

## 3. 用 cURL 验证

### 列出模型

```bash
curl https://sub2api.lingshichat.top/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

### Chat Completions

```bash
curl https://sub2api.lingshichat.top/v1/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "grok-4.5",
    "messages": [
      {"role": "user", "content": "ping"}
    ]
  }'
```

### Responses API（Grok Build 推荐）

```bash
curl https://sub2api.lingshichat.top/v1/responses \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{
    "model": "grok-4.5",
    "input": "ping"
  }'
```

若返回正常 JSON / 流式数据，说明 Key 与分组可用。

## 5. OpenAI SDK 示例

### Python

```python
from openai import OpenAI

client = OpenAI(
    base_url="https://sub2api.lingshichat.top/v1",
    api_key="YOUR_API_KEY",
)

resp = client.chat.completions.create(
    model="grok-4.5",
    messages=[{"role": "user", "content": "你好"}],
)
print(resp.choices[0].message.content)
```

### Node.js

```javascript
import OpenAI from 'openai';

const client = new OpenAI({
  baseURL: 'https://sub2api.lingshichat.top/v1',
  apiKey: 'YOUR_API_KEY',
});

const resp = await client.chat.completions.create({
  model: 'grok-4.5',
  messages: [{ role: 'user', content: '你好' }],
});

console.log(resp.choices[0].message.content);
```

## 常见失败排查

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| `401` | Key 错误或未带 Bearer | 检查 `Authorization` 头 |
| `403` / 分组错误 | Key 未绑定可用分组 | 在控制台检查密钥分组 |
| `model_not_found` | 模型名不对或分组无该模型 | 使用 `grok-4.5`，或先 `GET /v1/models` |
| 超时 / 连接失败 | 网络或域名写错 | 确认使用 `sub2api.lingshichat.top` |
