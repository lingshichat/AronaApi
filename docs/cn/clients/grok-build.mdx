# Grok Build 配置（Grok 4.5）

本文说明如何让 [Grok Build](https://grok.com) CLI/TUI 通过中转站使用 `grok-4.5`。

## 推荐结论

| 项 | 推荐值 |
| --- | --- |
| Base URL | `https://sub2api.lingshichat.top/v1` |
| model | `grok-4.5` |
| api_backend | `responses` |
| supports_backend_search | `true` |
| web_search 模型 | 同样指向 `grok-4.5`（单独 model 段） |
| **context_window（实用）** | **`200000`（约 200k）** |
| context_window（硬上限） | 模型可到 `500000`（约 500k），但不建议日常拉满 |
| **数据训练 / 数据用于改进模型** | **关闭（必做）** |

<Warning>
  **Grok 4.5 上下文：上限 500k，但超过约 200k 会明显降智。**  
  模型技术上可接受约 **500k** tokens 的上下文，但有效推理质量在 **约 200k 之后会明显下降**（答非所问、漏改文件、工具调用变差等）。  
  编程代理场景请把客户端 `context_window` 设为 **`200000`**，让自动压缩在质量拐点前触发；不要为了「用满 500k」而长期挂超长会话。
</Warning>

<Warning>
  **务必关闭 Grok Build 的「数据训练 / 数据用于改进模型」开关。**  
  走中转站时，对话仍可能经过 Grok 客户端的隐私与数据保留策略。配置模型前先用 `/privacy` 确认状态为 **不用于训练 / 不共享以改进模型**。详情见下方 [关闭数据训练（必做）](#关闭数据训练必做)。
</Warning>

<Tip>
  聊天正常但 `web_search` 报 `402 Payment Required` / `spending-limit` 时，通常是 **搜索走了独立模型且未开 backend search**。把对话模型和 `web_search` 都指到中转站的 `grok-4.5`，并设置 `supports_backend_search = true`。
</Tip>

## 0. 关闭数据训练（必做）

使用中转站 **不代表** 可以跳过客户端隐私设置。请在 Grok Build 里明确关掉数据训练相关开关：

1. 启动 Grok Build（TUI）。
2. 输入 slash 命令：

```text
/privacy
```

3. 查看 **privacy / data-retention（隐私与数据保留）** 状态。
4. 将 **数据训练 / 用于改进模型 / data sharing for training** 相关选项 **全部关闭（Off）**。
5. 若界面提供切换，再执行一次 `/privacy` 确认已是关闭状态。

也可以打开设置面板核对：

```text
/settings
```

在设置中找到 Privacy / Data controls（或同等命名），确认「Improve the model / 用你的数据训练」类开关为 **关闭**。

### 建议一并关闭的客户端遥测（可选）

编辑 `~/.grok/config.toml`：

```toml
[features]
telemetry = false
```

这与「关闭数据训练」不是同一开关，但可减少匿名使用数据上报。

<Note>
  `/privacy` 管的是 **隐私与数据保留 / 是否用于训练**；`[features].telemetry` 管的是 **匿名使用遥测**。两边都建议按需关闭。  
  若你仍登录了 xAI / X 账号，还可在 X：Settings → Privacy and safety → Grok 中关闭「Allow posts/interactions to be used for training」类选项（与 Grok Build 客户端开关互补）。
</Note>

## 1. 准备 API Key

1. 打开 [控制台](https://sub2api.lingshichat.top) 创建 API Key
2. 用环境变量保存（推荐）：

```powershell
# Windows PowerShell
$env:ARONA_API_KEY = "sk-xxxxxxxx"
# 可选：写入用户环境变量，新开终端生效
[System.Environment]::SetEnvironmentVariable("ARONA_API_KEY", "sk-xxxxxxxx", "User")
```

```bash
# macOS / Linux
export ARONA_API_KEY="sk-xxxxxxxx"
```

## 2. 编辑 `~/.grok/config.toml`

Windows 路径：

```text
C:\Users\<你的用户名>\.grok\config.toml
```

macOS / Linux：

```text
~/.grok/config.toml
```

### 推荐完整配置

```toml
# 默认模型与 web_search 都走中转站 grok-4.5
[models]
default = "arona-grok45"
web_search = "arona-web-search"
# 可选：默认思考强度
# default_reasoning_effort = "high"

[model.arona-grok45]
model = "grok-4.5"
name = "Arona Grok 4.5"
description = "via sub2api.lingshichat.top"
base_url = "https://sub2api.lingshichat.top/v1"
env_key = "ARONA_API_KEY"
api_backend = "responses"
# 实用窗口：约 200k。模型硬上限约 500k，但超 200k 易降智
context_window = 200000
supports_backend_search = true

[model.arona-web-search]
model = "grok-4.5"
name = "Arona Grok 4.5 Web Search"
base_url = "https://sub2api.lingshichat.top/v1"
env_key = "ARONA_API_KEY"
api_backend = "responses"
context_window = 200000
supports_backend_search = true
```

### 方案 B：覆盖内置 `grok-4.5` 段

如果你更想直接用模型名 `grok-4.5`：

```toml
[models]
default = "grok-4.5"
web_search = "grok-4.5"

[model.grok-4.5]
base_url = "https://sub2api.lingshichat.top/v1"
env_key = "ARONA_API_KEY"
api_backend = "responses"
context_window = 200000
supports_backend_search = true
```

<Note>
  覆盖内置模型时，只需写你要改的字段；其余继承 Grok 默认。但 **web_search 仍建议显式写 `supports_backend_search = true`**。
</Note>

### 方案 C：用 endpoints 统一改出口

```toml
[endpoints]
models_base_url = "https://sub2api.lingshichat.top/v1"

[models]
default = "grok-4.5"
web_search = "grok-4.5"

[model.grok-4.5]
env_key = "ARONA_API_KEY"
api_backend = "responses"
supports_backend_search = true
context_window = 200000
```

设置 `models_base_url` 后，Grok 会用 API Key 鉴权拉取 `/v1/models`，通常不再依赖 `grok login`。

## 3. 启动并切换模型

```bash
# 查看模型列表
grok models

# 直接指定模型启动
grok -m arona-grok45

# 会话内切换
/model arona-grok45
```

## 4. 验证

### 命令行

```bash
grok -p "用一句话介绍你自己" -m arona-grok45
```

### 手动 curl（对照）

```bash
curl https://sub2api.lingshichat.top/v1/responses \
  -H "Authorization: Bearer $ARONA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grok-4.5",
    "input": "ping"
  }'
```

### web_search 自检

在 Grok Build 里要求模型：

```text
请用 web_search 搜索今天的公开新闻标题，给我 3 条。
```

- 成功：返回检索摘要
- 失败且 402 / spending-limit：检查 `web_search` 模型段与 `supports_backend_search`

## 字段说明

| 字段 | 含义 |
| --- | --- |
| `model` | 发给上游的模型 ID，这里用 `grok-4.5` |
| `base_url` | OpenAI 兼容根路径，需含 `/v1` |
| `env_key` | 从环境变量读 API Key，避免明文写进配置 |
| `api_backend` | `responses` / `chat_completions` / `messages` |
| `supports_backend_search` | 允许服务端 backend 搜索；web_search 需要 |
| `context_window` | 触发自动压缩的上下文估计值。**推荐 `200000`**（质量拐点）；模型硬上限约 `500000`，但超过约 200k 会降智 |

## 上下文配置（务必读）

| 档位 | tokens | 建议 |
| --- | --- | --- |
| **推荐工作区** | **≤ ~200k** | 日常编程代理保持在此区间，质量最稳 |
| 硬上限 | ~500k | 模型可接收，但 **超过约 200k 会明显降智** |
| 客户端 `context_window` | 推荐写 `200000` | 让自动压缩在降智前触发，而不是拖到 500k |

实践建议：

1. 配置里写 **`context_window = 200000`**，不要默认写成 `500000`「占满规格」。
2. 长任务中途开新会话 / 手动压缩，避免对话无限堆叠。
3. 不要一次性塞入整个巨型 monorepo；按任务范围缩小 `@` / 文件上下文。
4. 仅当你 **明确需要** 超长窗口且能接受质量下滑时，才临时调高到 `500000`。

## 可选：Chat Completions 后端

若 Responses 路径异常，可临时改用：

```toml
api_backend = "chat_completions"
```

多数基础对话可用；**web_search 仍建议 responses + supports_backend_search**。

## 常见问题

### 1. 模型列表里没有 arona-grok45

- 确认 `config.toml` 语法正确（TOML 段名、引号）
- 重新打开终端 / 重启 Grok Build
- 运行 `grok models` 查看是否加载到自定义模型

### 2. 401 Unauthorized

- 环境变量是否在 **当前终端** 生效
- Key 是否复制完整
- 是否误把 Key 写进了错误字段

### 3. 能聊天但不能 web_search

按下面清单逐项核对：

1. `[models].web_search` 是否指向自定义中转模型
2. 该模型段 `base_url` 是否为中转站
3. 是否 `supports_backend_search = true`
4. 对话模型与搜索模型是否都指向可用的 `grok-4.5`

### 4. 上下文很快被压缩 / 上下文很长后开始降智

- **降智优先**：把 `context_window` 设为 **`200000`**，让客户端在约 200k 附近压缩，而不是拖到 500k。
- Grok 4.5 **硬上限约 500k**，但 **超过约 200k 质量会明显下降**；`context_window` 只影响客户端压缩策略，不是「越大越聪明」。
- 若你误把窗口设成 `500000` 且会话已经很长：开新会话、删掉无关历史、缩小挂载的文件范围。
- 仅在极少数需要硬塞超长材料的场景才临时用 `500000`，用完改回 `200000`。

### 5. 如何确认数据训练已关闭？

1. 在 TUI 执行 `/privacy`，确认 training / data-retention 为关闭  
2. 再执行 `/settings`，核对 Privacy / Data controls  
3. 可选：`config.toml` 中 `[features] telemetry = false`  
4. 若曾登录 xAI/X，到 X 的 Grok 隐私设置再关一遍账号侧训练开关  

**未关闭时不要把敏感代码 / 密钥贴进对话。**
