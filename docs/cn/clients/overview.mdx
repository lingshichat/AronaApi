# 客户端配置总览

本节只围绕 **Grok 4.5** 讲解主流编程客户端如何对接中转站。

## 统一参数

无论哪个客户端，先记住这 3 个值：

```text
Base URL : https://sub2api.lingshichat.top/v1
Model    : grok-4.5
API Key  : 控制台创建的密钥
上下文   : 硬上限 ~500k；实用请 ≤ ~200k（超 200k 易降智）
```

<Warning>
  **Grok 4.5 支持约 500k 上下文，但超过约 200k 会降智。**  
  客户端 `context_window` / `contextWindow` 建议写 **`200000`**，而不是默认 `500000`。详见 [Grok Build · 上下文](/cn/clients/grok-build#上下文配置务必读) 与 [模型支持](/cn/guides/models#grok-45-上下文重要)。
</Warning>

<Note>
  Claude Code 的 `ANTHROPIC_BASE_URL` 一般 **不要** 带 `/v1`，因为 SDK 会自己拼 `/v1/messages`。详见 [Claude Code 配置](/cn/clients/claude-code)。
</Note>

## 支持的协议

中转站（sub2api）对 Grok 分组主要提供：

| 协议 | 路径 | 典型客户端 |
| --- | --- | --- |
| OpenAI Chat Completions | `POST /v1/chat/completions` | 通用 SDK、OpenCode、Pi、部分 CLI |
| OpenAI Responses | `POST /v1/responses` | Grok Build、Codex、Pi（可选） |
| Anthropic Messages | `POST /v1/messages` | Claude Code |
| 模型列表 | `GET /v1/models` | 全部 |

## 配置前自检

```bash
# 1) Key 是否可用
curl https://sub2api.lingshichat.top/v1/models \
  -H "Authorization: Bearer $ARONA_API_KEY"

# 2) 对话是否可用
curl https://sub2api.lingshichat.top/v1/chat/completions \
  -H "Authorization: Bearer $ARONA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"grok-4.5","messages":[{"role":"user","content":"ping"}]}'
```

两项都成功后，再改客户端配置，能少踩很多坑。

## 当前文档范围

- ✅ Grok 4.5
- ✅ Grok Build / Codex / Claude Code / OpenCode / Pi Agent
- ⏳ 其他模型与客户端后续再补

如果你有额外模型别名（例如 `grok-latest`），以控制台 `GET /v1/models` 返回结果为准。
