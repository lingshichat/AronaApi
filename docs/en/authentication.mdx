# Authentication

Arona API uses API key authentication. Every gateway request needs a valid key.

## Auth method

Send:

```bash
Authorization: Bearer YOUR_API_KEY
```

## Get an API key

1. Sign in to the [console](https://sub2api.lingshichat.top)
2. Open **API Keys**
3. Create a key and store it offline

<Warning>
  Never put API keys in frontend pages, public repos, screenshots, or chat logs. Prefer environment variables.
</Warning>

## Recommended environment variables

| Variable | Use |
| --- | --- |
| `ARONA_API_KEY` | Generic clients / scripts |
| `XAI_API_KEY` | Some Grok-compatible clients |
| `OPENAI_API_KEY` | OpenAI SDK / Codex-style clients |
| `ANTHROPIC_AUTH_TOKEN` | Claude Code |

PowerShell:

```powershell
$env:ARONA_API_KEY = "sk-xxxxxxxx"
```

bash:

```bash
export ARONA_API_KEY="sk-xxxxxxxx"
```

## Request example

```bash
curl https://sub2api.lingshichat.top/v1/chat/completions \
  -H "Authorization: Bearer $ARONA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grok-4.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

## Keys and groups

API keys are usually bound to one or more **groups**. A group controls:

- Available models (for example `grok-4.5`)
- Upstream platform type (Grok / OpenAI / Claude, …)
- Quota, concurrency, and billing rules

If the model is unavailable or you get a permission error, check:

1. Key validity
2. Whether the key’s group includes `grok-4.5`
3. Account balance / subscription

## Security tips

- Use separate keys per client so you can revoke independently
- Rotate keys periodically
- Prefer `env_key` over pasting secrets into committed config files
