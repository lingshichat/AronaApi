# Client overview

This section covers connecting **Grok 4.5** through the gateway for popular coding clients.

## Shared settings

Remember these three values:

```text
Base URL : https://sub2api.lingshichat.top/v1
Model    : grok-4.5
API Key  : key created in the console
Context  : hard max ~500k; keep practical ≤ ~200k (quality drops past 200k)
```

<Warning>
  **Grok 4.5 supports ~500k context, but quality drops past ~200k.**  
  Prefer `context_window` / `contextWindow` = **`200000`**, not a default of `500000`. See [Grok Build · Context](/en/clients/grok-build#context-window-important) and [Models](/en/guides/models#grok-45-context-important).
</Warning>

<Note>
  For Claude Code, `ANTHROPIC_BASE_URL` usually must **not** include `/v1`, because the SDK appends `/v1/messages`. See [Claude Code](/en/clients/claude-code).
</Note>

## Supported protocols

For Grok groups, the gateway typically exposes:

| Protocol | Path | Typical clients |
| --- | --- | --- |
| OpenAI Chat Completions | `POST /v1/chat/completions` | SDKs, OpenCode, Pi, many CLIs |
| OpenAI Responses | `POST /v1/responses` | Grok Build, Codex, Pi (optional) |
| Anthropic Messages | `POST /v1/messages` | Claude Code |
| Model list | `GET /v1/models` | All |

## Pre-flight checks

```bash
# 1) Key works
curl https://sub2api.lingshichat.top/v1/models \
  -H "Authorization: Bearer $ARONA_API_KEY"

# 2) Chat works
curl https://sub2api.lingshichat.top/v1/chat/completions \
  -H "Authorization: Bearer $ARONA_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"grok-4.5","messages":[{"role":"user","content":"ping"}]}'
```

Fix API issues first, then edit client configs.
