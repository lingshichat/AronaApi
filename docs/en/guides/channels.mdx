# Channels and groups

For end users, what matters is:

1. Your **API key**
2. The **group** bound to that key
3. The **models** the group exposes (for example `grok-4.5`)

## Mental model

```text
Client  --API Key-->  Arona gateway  --group routing-->  upstream account pool
```

You only need to:

- Create a key in the console
- Confirm the key can call `grok-4.5`
- Set Base URL to `https://sub2api.lingshichat.top/v1`

## Console checks

- Key enabled
- Group includes Grok models
- Balance / subscription OK
- No rate-limit or risk blocks

## Admins (brief)

Admins configure upstream channels, pools, mappings, and load balancing. End users usually do not need those details.
