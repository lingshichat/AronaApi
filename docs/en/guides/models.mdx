# Models

## Recommended

| Model ID | Notes | Suggested clients |
| --- | --- | --- |
| `grok-4.5` | Primary chat / coding model | Grok Build, Codex, Claude Code, OpenCode, Pi |

Always trust the model list returned for **your** key.

## Grok 4.5 context (important)

| Item | Value |
| --- | --- |
| Hard max | ~**500k** tokens |
| **Quality soft limit** | ~**200k** tokens |
| Client setting | `context_window` / `contextWindow` = **`200000`** |

<Warning>
  **Grok 4.5 accepts ~500k context, but quality drops past ~200k.**  
  Keep coding agents in a **≤200k** working set and let the client compact earlier. Do not run forever-long sessions just to fill the 500k window. See [Grok Build · Context](/en/clients/grok-build#context-window-important).
</Warning>

## List models

```bash
curl https://sub2api.lingshichat.top/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"
```

```json
{
  "object": "list",
  "data": [
    {
      "id": "grok-4.5",
      "object": "model",
      "owned_by": "xai"
    }
  ]
}
```

## Models and groups

- The API key **group** decides which models you see
- The same name may map to different upstream pools
- On `model_not_found`, check the group first

## Call example

```bash
curl https://sub2api.lingshichat.top/v1/chat/completions \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "grok-4.5",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```
